XML

XML quindi non è l'evoluzione di HTML. Bensì possiamo definirlo come un metalinguaggio che ha lo scopo di rappresentare contenuti testuali organizzati in modo gerarchico ed il suo punto di forza è la sua capacità di rappresentare il contenuto del file in un formato compatibile con qualsiasi sistema hardware e software in quando si tratta di un semplice file in formato testo.

In particolare, XML non è:

  • un linguaggio di presentazione come HTML: il markup XML definisce il significato dei dati e non dice nulla sullo stile dei dati;
  • un linguaggio di programmazione come Java: un documento XML non è in grado di eseguire calcoli;
  • un protocollo di trasporto come http: XML non trasferisce i dati attraverso la rete;
  • un DBMS come Acess o MySql: XML non memorizza e recupera i dati.
